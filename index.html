1) Use Case tổng (tách vai – khuyến nghị)

Mục tiêu & Actors

Mục tiêu: Chuẩn hóa quy trình từ Program/Version → Training Request → Lập lịch (Planner phối hợp Instructor) → Region Review → Công bố & Export.

Actors: PO, Requester, Planner, Instructor, Region Coordinator.

ProgramVersion = Published
Request = Approved
Class = Published

Tiền điều kiện

  • Có tối thiểu 01 ProgramVersion = Published.
  • Danh mục miền/campus/phòng/ngày nghỉ đã thiết lập cơ bản.

Luồng chính (B1 → B10)

  1. B1 · PO: tạo Program (intro, outcomes, audience, tổng giờ, topic) → Program = Draft.
  2. B2 · PO: tạo & Publish ProgramVersion (vd. v1.0) → ProgramVersion = Published.
  3. B3 · Hệ thống/PO: công bố Catalog read‑only cho BU.
  4. B4 · Requester: gửi Training Request (audience, mục tiêu, miền/campus, khung giờ mong muốn, ràng buộc) → Submitted.
  5. B5 · Planner: review & Approve request → Approved.
  6. B6 · Planner: tạo Class từ ProgramVersion Published → Planned.
  7. B7 · Planner: Generate timeline (pattern, skip holiday), check trùng cơ bản (GV/giờ) → Timeline (Proposed).
  8. B8 · Planner + Instructor: Assign & Confirm giảng viên.
  9. B9 · Region: Approve ràng buộc miền (holiday, phòng, sự kiện) → Region‑Approved.
  10. B10 · Planner: Publish & Export CSV/Excel (ICS tùy chọn) → thông báo các bên → Class = Published.

2) Swimlane (Actor | Hành động | Kết quả)

P1. Chuẩn bị & công bố chương trình

Actor Hành động Kết quả
PO Tạo Program (mô tả, outcomes, audience, tổng giờ; thêm Topic) Program = Draft
PO Tạo & Publish ProgramVersion (v1.0) ProgramVersion = Published
Hệ thống/PO Công bố Catalog (read‑only) cho BU/Requester BU tham khảo nội dung/targets

P2. Yêu cầu mở lớp

Actor Hành động Kết quả
Requester (BU) Submit Training Request (đối tượng, mục tiêu, miền/campus, khung giờ, ràng buộc) Request = Submitted
Planner Review nhanh, trao đổi nếu cần → Approve Request = Approved

P3. Lập lớp & timeline

Actor Hành động Kết quả
Planner Tạo Class từ ProgramVersion Published; nhập region/campus, start date, capacity Class = Planned
Planner Generate timeline (pattern, skip holiday); check trùng cơ bản (GV/giờ) Timeline (Proposed), không trùng giờ GV
Planner Assign Instructor theo skill/availability Class gán GV, chờ xác nhận
Instructor My Classes → Confirm; góp ý nhịp dạy trong phạm vi timeline Instructor = Confirmed

P4. Duyệt ràng buộc miền

Actor Hành động Kết quả
Region Coordinator Rà holiday/sự kiện campus/phòng → Approve by Region Class = Region‑Approved

P5. Công bố & vận hành

Actor Hành động Kết quả
Planner Publish lớp; Export CSV/Excel (ICS tùy chọn); thông báo Requester/Instructor Class = Published; lịch chia sẻ, file tải được
Instructor Giảng dạy theo lịch; nếu phát sinh, đề nghị Reschedule qua Planner Lịch được cập nhật & thông báo các bên
